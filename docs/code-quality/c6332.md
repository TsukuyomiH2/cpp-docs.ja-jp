# <a name="c6332"></a>C6332

> 警告 C6332: 無効なパラメーター: dwFreeType パラメーターとして0を渡す \<function> ことは許可されていません。 この呼び出しはエラーになります。

この警告は、無効なパラメーターが VirtualFree または VirtualFreeEx に渡されていることを示します。 VirtualFree と VirtualFreeEx のいずれについても、dwFreeType パラメーターに 0 を設定することは拒否されます。 dwFreeType パラメーターには、MEM_DECOMMIT または MEM_RELEASE を指定できます。 ただし、同じ呼び出しで 値 MEM_DECOMMIT と 値 MEM_RELEASE を同時に使用することはできません。 また、VirtualFree 関数の戻り値が無視されないことを確認してください。

## <a name="example"></a>例

次のコードでは、無効なパラメーターが VirtualFree 関数に渡されるために、この警告が発生します。

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, 0 );
  // code ...
}
```

この警告を解決するには、次のコードに示すように VirtualFree 関数の呼び出しを変更します。

```cpp
#include <windows.h>
#define PAGELIMIT 80

DWORD dwPages = 0;  // count of pages
DWORD dwPageSize;   // page size

VOID f( VOID )
{
  LPVOID lpvBase;            // base address of the test memory
  BOOL bSuccess;
  SYSTEM_INFO sSysInfo;      // system information

  GetSystemInfo( &sSysInfo );
  dwPageSize = sSysInfo.dwPageSize;

  // Reserve pages in the process's virtual address space
  lpvBase = VirtualAlloc(
                         NULL,                // system selects address
                         PAGELIMIT*dwPageSize, // size of allocation
                         MEM_RESERVE,
                         PAGE_NOACCESS );
  if (lpvBase)
  {
    // code to access memory
  }
  else
  {
    return;
  }

  bSuccess = VirtualFree( lpvBase, 0, MEM_RELEASE );
  // code ...
}
```

VirtualAlloc と VirtualFree の使用には、メモリ リークと例外に関して多くの落とし穴があります。 これらのリークと例外の問題を完全に回避するには、C++ の標準テンプレート ライブラリ (STL) によって提供される機構を使用します。 これには、 [shared_ptr](../standard-library/shared-ptr-class.md)、 [unique_ptr](../standard-library/unique-ptr-class.md)、および [ベクター](../standard-library/vector.md)が含まれます。 詳細については、「 [スマートポインター](../cpp/smart-pointers-modern-cpp.md) 」と「 [C++ 標準ライブラリ](../standard-library/cpp-standard-library-reference.md)」を参照してください。
